Procédure de Reproduction Spécifique

1. Prérequis et Installation
- Installer Ubuntu Server (VM1) et une VM de test (VM2 : Ubuntu/Kali).
- Sur VM1, installer les paquets :
  sudo apt update
  sudo apt install auditd audispd-plugins rsyslog logrotate python3 python3-flask -y

2. Configuration de l'Audit (auditd)
- Copier le fichier `security.rules` généré dans `/etc/audit/rules.d/` :
  sudo cp config/security.rules /etc/audit/rules.d/
- Recharger et redémarrer :
  sudo augenrules --load
  sudo systemctl restart auditd
- Vérifier les règles avec :
  sudo auditctl -l

3. Lancement des Scripts de Surveillance (Locaux)
- Donner les droits d'exécution :
  chmod +x scripts/*.sh
- Exécuter manuellement pour un test :
  sudo ./scripts/security_monitor.sh
  sudo ./scripts/audit_report.sh

4. Configuration de l'Application Dashboard Python (Interface)
- Se placer dans le dossier de l'application :
  cd app/
- Démarrer le serveur Flask avec droits sudo (pour lire les logs d'audit) :
  sudo python3 app.py
- Accéder au tableau de bord via un navigateur depuis l'hôte : http://[IP_VM1]:5000/

5. Lancement des Scénarios de Test
- Scénario 1 (Brute force depuis VM2) : ssh fakeuser@[IP_VM1]
- Scénario 2 (Sudo sur VM1) : sudo ls /root
- Scénario 3 (Modification fichier): sudo touch /etc/passwd
- Scénario 4 (Changement mdp) : passwd

6. Vérification des Résultats 
- Consulter le tableau de bord web créé en Python.
- Vérifier localement dans `/var/log/security_alerts.log`.
- Vérifier via la commande `sudo ausearch -k [CLE]`.
